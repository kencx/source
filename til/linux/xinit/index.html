Xinit
29 Dec 2022 • 2 mins

When debugging GUI issues, you can switch to a different tty with Ctrl+Alt+F{1-7}.

xinit is used to start window managers for desktop environments. It starts a Xorg (or X) display server on Linux systems.

## Installation

  1. Install the required packages
$ pacman -S xorg-server xorg-xinit xterm

xterm is necessary for xorg to work properly on a fresh install.

  1. Install the required video drivers
# identity your graphics card
$ lspci -v | grep -A1 -e VGA -e 3D

# install appropriate driver
$ pacman -S xf86-video[-intel|-amdgpu]

## Configuration

  1. Copy default xinitrc to home directory
$ cp /etc/X11/xinit/xinitrc ~/.xinitrc
  1. Edit the file by replacing the default programs with your desired startup commands
# .xinitrc
...
sxhkd &
exec bspwm
  1. Run Xorg with startx. Do not use exec startx.
$ startx
  1. To autostart X at login, include the following in the user’s .bash_profile (or end of .zshrc),
# .bash_profile
if [[ -z $DISPLAY ]] && [[ $(tty) = /dev/tty1 ]]; then
	exec startx
fi

To redirect all stdout and stderr output from the Xorg session to a log file .xorg.log, use the following startx command instead:

startx -- -keeptty > ~/.xorg.log 2>&1
  1. If using a display manager like LightDM, you must manage sessions startup with ~/.xprofile or run xinitrc as a session
# .xprofile
sxhkd &

If using ly, xinitrc support is enabled of out the box.
