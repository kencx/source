Pushing to Pull Request
8 Dec 2024 • 1 min

Say a contributor has submitted a PR to your project. Before merging, you would like to perform a minor change to their PR.

## Requirements

  • The contributor forked your project
  • The contributor made changes on a feature branch
  • The contributor submitted a pull request from their fork to origin/master
  • The contributor has explicitly allowed you to push to their branch. This is usually the case if you are a maintainer.

## Steps

Assuming the contributor is johnadams.

  1. Add the contributor’s fork as a remote
$ git remote add johnadams https://github.com/johnadams/repo.git
$ git remote -v
  1. Fetch their list of branches
$ git fetch johnadams
  1. Create a new branch johnadams-feature from the branch they created the PR from
$ git checkout -b johnadams-feature johnadams/feature
  1. Make the necessary changes.
  2. Push the changes back to the PR by pushing to their branch
$ git push johnadams johnadams-feature:feature

Take care of the push arguments

$ git push <repository> <refspec>

where

  • <repository> is the destination remote
  • <refspec> is the source object AND destination ref/branch to update, in the format: <src>:<dest-ref>
